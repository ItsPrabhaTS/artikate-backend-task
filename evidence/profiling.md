# Section 1 - profiler evidence

Dataset: customer id=1 with 250 orders (SQLite, local).
Generated by `python manage.py profile_summary`.

| Endpoint | SQL queries | Wall time |
|---|---|---|
| `/api/orders/summary-broken/` | 751 | 830 ms |
| `/api/orders/summary/` (fixed) | 2 | 74 ms |

Query count: 751 -> 2; wall time ~11x faster on localhost.

The gap widens dramatically with real network latency: at ~1ms of
round-trip time per query, the broken endpoint pays that cost per
order, the fixed one pays it twice in total.

django-silk recorded the same requests (broken: 751 queries, fixed: 2 queries) - browse them at `/silk/` with the dev server running.
